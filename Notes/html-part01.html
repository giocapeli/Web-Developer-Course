HTML Basic - Introdution

Hypertext Markup Language

O que é HTML?

HTML não é uma linguagem de programação, mas sim uma markup language, ou, linguagem de marcação que é estruturada para dar significado e organizar as informações dos sites. Ela é construída a partir de elements formados por um content inserido dentro de uma tag. Ela pode trabalhar junto a outras linguagens como CSS e Javascript, que por sua vez serão usadas respectivamente para definir a aparência da página e suas funcionalidades.

HTML elements

O que é um elemento?

O elemento é um conjunto formado por uma tag e seu content, podendo ou não possuir attributes.

Exemplo:

1 <tag>content</tag>

A opening tag fica entre símbolos de "<" e ">" e indica o início do element, a closing tag também fica entre "<" e ">" porém com uma "/" para indicar o fechamento do element.

Attributes

O que são atributos?

Os attributes são como detalhamentos, informações passadas dentro da tag para que ela se comporte de determinada maneira ou receba determinada característica. São compostos por um name e um value.

Exemplo:

1 <p class="classValue" id="idValue">Olá mundo.</p>

Cada tag pode possuir diversos attributes que depois podem ser facilmente conectados a funções em CSS ou Javascript.

Basic HTML document

Estrutura básica do HTML

A estrutura básica do HTML é composta pelas tags <!DOCTYPE html>, <html>, <head> e <body>. São quase sempre presente a <meta charset> e o <title>.

Exemplo:

  • !DOCTYPE html: Essa vai ser sempre a primeira linha do documento, ela não é uma tags mas sim uma instrução ao navegador de que o código que vem a seguir é um html. É obrigatório.
  • html: É o element que envolve todo conteúdo, visível ou não, da página. Também chamado de root element.
  • head: Serve como container de todo content ou instrução que não ficará visível na página principal, como o título da página, referências aos arquivos CSS, keywords e descrições para mecanismos de busca.
  • meta: As metatags aparentam não ter nenhum efeito na página em si, mas fornecem informações para o browser e para buscadores. A quase sempre utilizada é a charset="utf-8" que indica ao browser que o documento utiliza o UTF-8, um padrão de código que inclui a grande maioria dos caracteres conhecidos. Problemas de não reconehcimento de acentuação, por exemplo, pode indicar a utilização de outros códigos, como o ASCII.
  • title: Define o título da página que aparecerá na aba do navegador, como padrão ao favoritar a página e título nos sistemas de busca do google.
  • body: É o element que armazena content visível, ou seja, o que o usuário irá visualizar ao entrar na página.

Comments

Adicionando comentários ao código

Os comentários não são exibidos na página e nem tem nenhuma função ao usuário mas servem como guia de leitura e entendimento do código para o desenvolvedor. Os comentários precisam estar inseridos entre <!-- e --> para funcionarem.

Exemplo:

</head>
<body>
Essa é minha primeira página! <!-- Aqui temos o conteúdo visível da página -->
Espero que gostem! <!-- Nenhum comentário será visível para o usuário -->
</body>
</html>

Resultado visível do exemplo:

Essa é minha primeira página! Espero que gostem!

Os comentários são de suma importância e sua qualidade informativa é proporcional à complexidade e tamanho do código, quanto mais difícil e longo ele é, mais os comentários ajudam tanto quem escreveu quanto quem irá modificar a entender o que foi feito. Os comentários também servem para fazer testes no código, ativando ou desativando elementos com rapidez sem deletar nenhuma linha. A maioria das IDEs possuem atalhos para transformar a seleção em comentário, caso o atalho ctrl+/ do Sublime 3 não esteja funcionando troque nas preferences por outro atalho, como ctrl+1.
